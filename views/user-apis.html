  • Create Domain
    PUT
    //:/domain
    API Specifications:
    Purpose: To create a new Domain for an user identified by appKey and secretKey.
    Request Headers: App Key and Secret Key are passed in headers to verify the user. 
    	{
	  "appKey": appKey,
	  "secretKey": secretKey
	}
    Request Payload: An user defined Domain Name is passed in the body. 
            {
          "domainName": domainName
        }
    Response: On creation of Domain user receives success response. 
            {
          "success": true,
          "message": "User Domain created"
        }
    Example: To create a domain named "My Domain" of an user verified by using "708ee372-b43e-4aef-b398-3b742c4647c1" as App Key and "d7ef50b0-37d9-4021-8863-94a1d0b665cd" as Secret Key
    request headers: 
                {
                  "appKey": "708ee372-b43e-4aef-b398-3b742c4647c1",
                  "secretKey": "d7ef50b0-37d9-4021-8863-94a1d0b665cd"
                }

request payload: 

                {
                  "domainName": "My Domain"
                }

response: 
                {
                  "success": true,
                  "message": "User Domain created"
                }

  • Put Data Headers to an entity
    PUT
    //:/domain/{domainName}/entity/{entityName}/data-headers
    API Specifications:
    Purpose: To add Data Headers of an entity of a domain.
    Request Headers: App Key and Secret Key are passed in headers to verify the user. 
    	{
	  "appKey": appKey,
	  "secretKey": secretKey
	}
    Request Payload: An user defined headers required for data are passed in the body. 
            {
          "headers": headers
        }
    Response: On creation of Headers user receives success response. 
            {
          "success": true,
          "message": "Headers created"
        }
    Example: To create headers with attribute name as "My attribute" and attribute type as "numeric" for an entity named "My Entity" of a domain named "My Domain" of an user verified by using "708ee372-b43e-4aef-b398-3b742c4647c1" as App Key and "d7ef50b0-37d9-4021-8863-94a1d0b665cd" as Secret Key
    url: //:/domain/My Domain/entity/My Entity/data-headers
request headers: 
                {
                  "appKey": "708ee372-b43e-4aef-b398-3b742c4647c1",
                  "secretKey": "d7ef50b0-37d9-4021-8863-94a1d0b665cd"
                }

request payload: 

                {
                  "headers": {
                    "headers": [
                      {
                        "index": 0,
                        "attributeName": "My attribute",
                        "attributeType": "numeric"
                      }
                    ]
                  }
                }

response: 
                {
                  "success": true,
                  "message": "Headers created"
                }
